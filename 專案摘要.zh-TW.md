## 📋 專案概述

`top-traffic-list-taiwan` 是一個從多個資料來源抓取並整合台灣網站流量排名清單的工具集。此專案為「民生數位服務韌性檢測」系統提供測試目標網站清單，確保檢測涵蓋台灣最重要的網站。

---

## 🎯 核心功能

### 1. 多來源資料抓取

從五個不同的資料來源抓取台灣網站排名：

| 資料來源 | 更新頻率 | 排名數量 | 特色 |
|---------|---------|---------|------|
| **Tranco List** | 每日 | ~12,000 筆 | 結合多來源，最具可靠性 |
| **Cloudflare Radar** | 每日 | 前 100 名 | 真實流量數據，包含分類資訊 |
| **AhrefsTop** | 每月 | 前 100 名 | 有機搜尋流量排名 |
| **SimilarWeb** | 每月 | 前 50 名 | 多來源數據整合 |
| **Semrush** | 每月 | 前 100 名 | 點擊流數據（clickstream） |

### 2. 資料整合與合併

**merge-lists.js**（253 行）
- 合併五個資料來源為統一格式
- 標準化網址處理（移除 `www.` 前綴，轉小寫）
- 保留各來源的 rank 資訊
- 智能排名比較（保留較小的 rank）
- 生成 `merged_lists_tw.json`（約 250KB）

**輸出格式：**
```json
{
  "website": "example.com",
  "url": "https://example.com",
  "rank": {
    "tranco": 123,
    "cloudflare": 45,
    "ahrefs": 67,
    "similarweb": 23,
    "semrush": 89
  }
}
```

### 3. 雲端服務商資料庫

**cloud_providers_tw.json**（503 行，約 12KB）

提供完整的雲端服務商識別資訊，用於判斷網站使用的雲端服務：

**A. 國際公有雲（有台灣節點）**
- Amazon Web Services
- Google Cloud
- Microsoft Azure
- Cloudflare
- Akamai

**B. 國際公有雲（無已知台灣節點）**
- IBM Cloud、Huawei Cloud、DigitalOcean、Fastly、Vercel

**C. 本地雲端服務商**
- 中華電信 HiCloud、遠傳電信 FETnet、台灣大哥大 Taiwan Mobile 等

**識別資訊：**
- ASN 號碼列表
- 組織關鍵字列表
- 備註資訊

### 4. 資料品質控制

**check-duplicates.js**
- 檢查合併清單中的重複項目
- 確保資料品質

## 🚀 使用方式

### 安裝依賴
```bash
npm install
```

### 抓取各來源資料
```bash
npm run tranco        # 抓取 Tranco List
npm run cloudflare    # 抓取 Cloudflare Radar（需要 API Token）
npm run ahrefs        # 抓取 Ahrefs
npm run similarweb    # 抓取 SimilarWeb
npm run semrush       # 抓取 Semrush
```

### 合併清單
```bash
npm run merge         # 合併所有來源為 merged_lists_tw.json
```

### 環境變數設定
建立 `.env` 檔案：
```
CLOUDFLARE_API_TOKEN=your_token_here
```

## 🔧 技術架構

### 技術棧
- **語言：** Node.js
- **依賴套件：**
  - `adm-zip`：處理 ZIP 檔案（Tranco List）
  - `dotenv`：環境變數管理

### 資料處理流程

```
1. 抓取各來源資料
   ├─ Tranco List（CSV/ZIP）
   ├─ Cloudflare Radar（API）
   ├─ Ahrefs（網頁爬蟲）
   ├─ SimilarWeb（網頁爬蟲）
   └─ Semrush（網頁爬蟲）

2. 轉換為統一 JSON 格式
   └─ 各來源產生對應的 JSON 檔案

3. 合併清單
   └─ merge-lists.js 合併所有來源
   └─ 生成 merged_lists_tw.json

4. 資料品質檢查
   └─ check-duplicates.js 檢查重複
```

## 🎯 專案目標與應用

### 主要目標
1. **提供測試目標清單**：為「民生數位服務韌性檢測」系統提供台灣重要網站清單
2. **多來源整合**：結合多個資料來源，提高清單的完整性和可靠性
3. **資料標準化**：統一不同來源的資料格式，便於後續處理

### 應用場景
- 作為 `web-resilience-test` 的測試目標清單
- 用於 `generate_statistic.js` 的排序依據
- 提供雲端服務商識別資料，用於判斷網站使用的雲端服務
